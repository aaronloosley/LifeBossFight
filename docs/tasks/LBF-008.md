# LBF-008: Multi-provider linking and account recovery strategy

**Tag:** LBF-008  
**Category:** Authentication

## Description
Plan how the app will behave when a user signs up with one method and later uses another. Account linking prevents duplicate user records and painful support incidents.

## Steps
1. Define linking rules for Google and Email/Password accounts.
2. Handle Firebase `account-exists-with-different-credential` scenarios.
3. Design recovery flows for users who forget which provider they used.
4. Document support and UX copy for provider mismatch cases.
