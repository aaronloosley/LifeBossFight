# LBF-016: Input validation and sanitization pass

**Tag:** LBF-016  
**Category:** Security

## Description
Review all user input surfaces so invalid, dangerous, or malformed data cannot propagate into storage, logs, reports, or rendered UI. This is essential for both reliability and OWASP-aligned hardening.

## Steps
1. Inventory all forms, query params, and import/upload surfaces.
2. Apply Zod validation and consistent normalization rules.
3. Escape or sanitize user-controlled rich text, filenames, and generated reports as needed.
4. Add targeted tests for invalid and edge-case inputs.
