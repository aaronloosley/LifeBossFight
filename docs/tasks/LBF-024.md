# LBF-024: Unit test expansion for business logic and auth helpers

**Tag:** LBF-024  
**Category:** Testing

## Description
Increase unit coverage around mission logic, auth state, validation, and any new shared utilities. This gives fast feedback on regressions and supports safer iteration.

## Steps
1. Identify untested helper modules with meaningful business logic.
2. Add focused unit tests for auth error mapping, profile persistence, and validation.
3. Mock Firebase modules cleanly to avoid brittle tests.
4. Track test gaps that still require integration or E2E coverage.
