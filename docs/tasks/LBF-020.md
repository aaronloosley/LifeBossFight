# LBF-020: Accessibility audit and remediation

**Tag:** LBF-020  
**Category:** Accessibility

## Description
Bring the product closer to WCAG expectations so users can complete stressful workflows regardless of device, vision, hearing, or motor differences. Accessibility is product quality, not just compliance.

## Steps
1. Audit semantic structure, labels, focus order, and keyboard access.
2. Verify color contrast, motion controls, and screen-reader announcements.
3. Fix issues on auth, mission, evidence, summary, and settings screens.
4. Add accessibility checks to QA and release acceptance.
