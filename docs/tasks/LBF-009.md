# LBF-009: Authenticated route protection

**Tag:** LBF-009  
**Category:** Authentication

## Description
Protect mission, evidence, settings, and other signed-in surfaces so anonymous users cannot access or mutate private data. Production access control must be consistent at both the UI and data layers.

## Steps
1. Add a shared auth state provider and loading state.
2. Wrap authenticated routes in a protected layout or guard.
3. Redirect signed-out users to `/login`.
4. Validate behavior for refreshes, expired sessions, and direct deep links.
