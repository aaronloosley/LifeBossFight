# LBF-018: Abuse prevention and rate-limiting strategy

**Tag:** LBF-018  
**Category:** Security

## Description
Plan controls for signup abuse, brute-force login attempts, automated scraping, and denial-of-wallet style traffic. Firebase reduces some burden, but production apps still need explicit anti-abuse decisions.

## Steps
1. Evaluate Firebase Auth throttling behavior for expected traffic.
2. Decide whether to enable App Check, CAPTCHA, or additional gateway protections.
3. Protect any future server endpoints or admin flows with rate limits.
4. Document incident handling for abuse spikes or suspicious account creation.
