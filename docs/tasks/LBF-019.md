# LBF-019: Audit logging for critical account actions

**Tag:** LBF-019  
**Category:** Security

## Description
Record important security-sensitive actions so support and engineering teams can investigate problems. Audit visibility becomes especially important once user data, password resets, and cloud sync are live.

## Steps
1. Define which events require audit coverage, such as login, logout, reset requests, and evidence deletion.
2. Choose where audit logs are stored and how long they are retained.
3. Ensure logs avoid leaking secrets or sensitive content unnecessarily.
4. Document who can access audit records and for what purpose.
