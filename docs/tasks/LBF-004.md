# LBF-004: Email/password registration flow

**Tag:** LBF-004  
**Category:** Authentication

## Description
Add first-party account creation so users can register without relying on Google sign-in. Production readiness requires a standard account path that supports email verification, recoverability, and non-Google users.

## Steps
1. Create a registration route and form using the existing TypeScript form stack.
2. Validate email, password strength, and password confirmation inputs.
3. Create the Firebase auth user and trigger verification email delivery.
4. Show clear success, error, and verification-required states.
