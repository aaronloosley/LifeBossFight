# LBF-022: Loading, empty, and error state system

**Tag:** LBF-022  
**Category:** UX

## Description
Users need calm, informative feedback when data is loading, unavailable, or absent, especially in emergency-adjacent workflows. This task turns edge states into a deliberate part of the experience.

## Steps
1. Inventory every screen state for loading, empty, error, and partial success.
2. Create reusable UI patterns for alerts, skeletons, and retry affordances.
3. Add tailored copy for auth, network, storage, and validation failures.
4. Test these states on slow networks and mobile devices.
