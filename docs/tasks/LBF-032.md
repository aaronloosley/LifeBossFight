# LBF-032: Preview deployment workflow

**Tag:** LBF-032  
**Category:** CI/CD

## Description
Set up predictable preview deployments so reviewers can validate changes before merge. Previews are especially valuable for auth, UI polish, and high-stress user flows.

## Steps
1. Define how pull requests map to preview deployments.
2. Decide whether previews use demo mode or a dedicated preview Firebase project.
3. Protect preview credentials and authorized domains appropriately.
4. Document how QA and stakeholders access preview builds.
