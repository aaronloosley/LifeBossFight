# LBF-011: Mission run cloud sync strategy

**Tag:** LBF-011  
**Category:** Data

## Description
The app currently stores data locally for the MVP, so production planning must define when and how mission runs sync to Firebase. A clear sync model is required for multi-device access, backups, and future collaboration.

## Steps
1. Decide which mission data remains local-first versus cloud-backed.
2. Define write timing, conflict rules, and offline reconciliation behavior.
3. Map mission run entities to Firestore collections using the existing schema guidance.
4. Document migration steps from local-only sessions to cloud-backed accounts.
