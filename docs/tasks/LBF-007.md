# LBF-007: Google sign-in production polish

**Tag:** LBF-007  
**Category:** Authentication

## Description
The repo already exposes Google sign-in, but the production flow needs domain, error, and fallback hardening. This task ensures the existing provider behaves reliably on every deployment surface.

## Steps
1. Configure authorized domains for local, preview, and production URLs.
2. Map popup and provider errors into friendly UX copy.
3. Handle cancelled popups, blocked popups, and unsupported browsers.
4. Verify first-time Google sign-in creates the expected user profile record.
