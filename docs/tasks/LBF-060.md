# LBF-060: Phone Stolen mission design and implementation

**Tag:** LBF-060  
**Category:** Mission Content

## Description
Add a digital and physical security mission for users whose phone has been stolen. This scenario combines urgency, account protection, and police or insurer reporting.

## Steps
1. Define the first critical actions for account lockout, SIM protection, device tracking, and remote erase decisions.
2. Add evidence and timeline prompts for theft location, transactions, and reference numbers.
3. Include branches for replacement access, MFA recovery, and bank alerting.
4. Test the mission for speed and clarity when the user may be using a borrowed device.
