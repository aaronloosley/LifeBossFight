# LBF-030: Offline and degraded-network resilience plan

**Tag:** LBF-030  
**Category:** Performance

## Description
Users may rely on the app during stressful real-world conditions, so network interruptions must be handled intentionally. This task defines what still works offline and how sync recovers when connectivity returns.

## Steps
1. List which features must remain functional without network access.
2. Decide how auth-required features behave when offline.
3. Add retry, caching, and recovery UX for failed cloud operations.
4. Test common flows in airplane mode and throttled-network conditions.
