# LBF-031: CI pipeline hardening and required checks

**Tag:** LBF-031  
**Category:** CI/CD

## Description
Define a robust continuous integration workflow that blocks unsafe changes before merge. Required checks keep quality standards enforceable instead of aspirational.

## Steps
1. Ensure lint, unit tests, build, and relevant E2E smoke tests run in CI.
2. Define which checks are required before merge.
3. Add caching, artifact retention, and clear failure reporting.
4. Document how contributors reproduce CI failures locally.
