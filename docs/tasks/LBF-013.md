# LBF-013: Firestore security rules implementation

**Tag:** LBF-013  
**Category:** Security

## Description
Secure Firestore so users can only access their own profile, mission, and evidence data unless a deliberate sharing model exists. Rules are a production-critical control and must match the application data model.

## Steps
1. Write owner-scoped rules for users, mission runs, evidence, and related collections.
2. Add field validation where practical to reduce malformed writes.
3. Test rules with the Firebase Emulator Suite or rules unit tests.
4. Publish reviewed rules to each Firebase environment.
