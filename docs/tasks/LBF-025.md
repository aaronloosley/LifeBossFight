# LBF-025: Integration tests for data and route guards

**Tag:** LBF-025  
**Category:** Testing

## Description
Add integration coverage for auth-aware flows that cross component and data boundaries. These tests catch issues that pure unit tests miss while remaining faster and more deterministic than full E2E runs.

## Steps
1. Test auth provider state transitions and protected-route redirects.
2. Validate profile creation, mission persistence, and evidence metadata logic.
3. Use emulator-backed or mocked Firebase services where appropriate.
4. Run these tests in CI on every pull request.
