# LBF-012: Evidence upload and storage synchronization

**Tag:** LBF-012  
**Category:** Data

## Description
Evidence is currently stored on-device by default, but production usage often requires optional cloud backup or sync. This task covers storage path design, upload controls, metadata tracking, and privacy expectations.

## Steps
1. Define whether evidence upload is optional, default-on, or plan-gated.
2. Design Firebase Storage paths keyed by authenticated user and mission run.
3. Persist upload metadata in Firestore and reconcile with local attachments.
4. Handle failed uploads, retries, and deletion flows cleanly.
