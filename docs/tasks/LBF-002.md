# LBF-002: Environment strategy and release tiers

**Tag:** LBF-002  
**Category:** Infrastructure

## Description
Formalize local, test, preview, staging, and production environments so deployments are predictable and isolated. This reduces the risk of shared credentials, contaminated test data, and unsafe release practices.

## Steps
1. Define which environments are required and what each one is used for.
2. Assign separate Firebase projects, domains, and secret sets to each environment.
3. Document promotion rules between environments and who can approve releases.
4. Update deployment documentation and environment naming conventions.
